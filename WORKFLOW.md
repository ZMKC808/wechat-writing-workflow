# 公众号写作工作流

## 每次启动必读

1. `系统/AGENTS.md` — 协作规则
2. `资料库/Style/AI飞升录写作技巧_v0.1.md` — 写作风格参考
3. `资料库/语料库/AI飞升录/` — 历史文章学习样本

## 工作流一：AI 自主学习

用户把链接扔进 `Inbox/` → 告诉 Codex 去学习

1. 读取 `Inbox/` 里的链接
2. 用 `wechat-article-to-markdown` 下载文章内容
3. 提取：核心观点、写作手法、标题公式、可复用素材
4. 整理归档到 `资料库/语料库/` 
5. 清空 `Inbox/`

## 工作流二：写稿

用户给：公众号链接 / 图片 / 热点选题

### 1️⃣ 调研

- **公众号链接** → 用 `wechat-article-to-markdown` 下载文章（含图片）
- **图片** → 读取图片内容，基于内容做同题扩展搜索更多资料
- **热点选题** → 用 `agent-reach` 查全网（X/小红书/GitHub/官网/Reddit）
- 多篇来源时自动整合，不单独写稿
- 调研产出放 `文章/{项目名}/过程/`

### 2️⃣ 写稿

- 在 `文章/` 下新建项目文件夹
- 按风格手册写 `文章.md`
- 参考历史文章的标题公式、开头结构、语言风格
- **更新 `Home.md` 的「当前在写」链接指向新文章的 `文章.md`**

### 3️⃣ 质量检查

- 用 `dbs-content` 做内容诊断
- 用 `dbs-ai-check` 检测 AI 味并修改
- 检查是否有"首先/其次/总而言之/值得注意的是"等 AI 高频词

### 4️⃣ 配图

- 优先用文章本身带的截图或用户给的图
- 没有合适截图时用 `ian-xiaohei-illustrations` 规划 shot list 后生成

### 5️⃣ 输出

- `文章.md` — 正文稿
- `发布版.md` — 含 base64 图片的发布格式
- `图片/` — 配图文件

### 6️⃣ 发布后

- 打开已发布文章链接，用 Camoufox 读取阅读数
- 记入 `复盘/周复盘模板.md`
- **更新 `Home.md` 的「当前在写」链接从 `文章.md` 改为 `发布版.md`**

## 工作流三：每周复盘

用户说"本周复盘"时：

1. 打开 `复盘/周复盘模板.md` 
2. 列出本周发布的文章及其阅读数
3. 列出本周 AI 学习了什么
4. 判断哪些是爆款（阅读明显高于均值）
5. 分析爆款原因（标题/选题/时机/福利）

## 重要禁止

- 不直接改写 `学习素材/` 里的原文
- 素材区文章只做学习参考和风格模仿
- 链接必须真实读取，不能凭印象总结
- 选题判断、标题方向、正文观点由用户决定
