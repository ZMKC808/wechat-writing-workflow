刚刚！黄仁勋三小时暴论：推理算力需求暴涨1万倍，龙虾 Agent 要来抢你的饭碗了

一、三小时暴论：AI 不再是“聊天工具”
--------------------

在很多人印象里，GTC 是芯片发布会，是工程师的“春晚”。

今年在圣何塞 SAP Center 现场，我最直观的感受只有一句话，更像是一场关于“谁掌控下一代劳动力”的宣战。

黄仁勋穿着那件熟悉的黑皮衣，一口气讲了三个多小时，从 CUDA 二十年、Vera Rubin、Groq LPU，到 OpenClaw、NemoClaw、Physical AI、Olaf 机器人，最后落在一个非常刺耳的问题上：

“未来，每一家 SaaS 公司都会变成智能体服务公司。那你准备好的 Agent 战略是什么？”这句话一出来，我脑子里第一个画面，就是最近深圳线下 Dragon 虾门店门口排队的人。

有人花几千块买一台小主机回家“养龙虾”，有人在办公室偷偷装 OpenClaw，让 AI 帮自己写日报、回邮件、跑表格，还有人干脆开了 AI 托管账号，让龙虾 24 小时在各个平台帮他发内容、回私信。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zm6HdKxqRwYM5Ib6iaNEbUVOPeUL4v8RSEOc270y3kQwhVkyL9Tt7vm6UMeevGB1RjUPOOG1zA1bicNdsDeqygiabFTOTgwv5SHRU/640?wx_fmt=png&from=appmsg)

很多人以为这只是一个互联网新玩具，顶多是生产力工具升级一下。但在黄仁勋眼里，这是一条清晰到不能再清晰的时间线：

\- 从 GeForce 到 CUDA，是“让 GPU 变得可编程”

\- 从 CUDA 到深度学习，是“算力飞轮”

\- 从深度学习到大模型，是“让机器能说话、能写”

\- 而从大模型到 Agent，是“让机器真的去干活，去接你的班”

如果说大模型在抢“脑力”，那龙虾 Agent 抢的，就是“时间”和“岗位”。

二、推理算力暴涨 1 万倍：真正涨价的是“你的时间”
--------------------------

过去两年，大家聊英伟达，更多是在看股价、看 Blackwell、看 H100 供不应求。这次 GTC，一个非常关键的转向，是黄仁勋几乎把“训练”一笔带过，把绝大多数篇幅都留给了推理。

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/JYWVQlsu9zkdC1VN3wwWkMdtUOLk0V2HauGo5D8GKOXhpSs4I7EibgPicBAxp0ngTNias3O28EQaAzy3ZWGyAs77T2qvOb3VViaSkJJEH6VtVOc/640?wx_fmt=png&from=appmsg)

他说了一句外行听着可能有点抽象的话：

“过去两年，推理所需计算量增加了约 10000 倍，使用量增加了约 100 倍。”

我当时脑子里立刻对上了国内这几个月的两个画面：一个是办公室里，越来越多人打开电脑第一件事，不是登录邮箱，而是先把 Cursor、龙虾、各种 Agent 面板拉出来，另一个是各大平台上，一大批“AI 托管账号”昼夜不停发内容、接私信、挂直播。

这些背后，消耗的都是推理算力。

以前我们用 AI，是一问一答，发一条消息，等一句回复，算一次账。现在的 Agent，动不动就要在内部思考几十步、上百步：

\- 先帮你读完一堆 PDF，再抽出关键条款

\- 然后去邮件里翻历史对话

\- 再连到浏览器、连到数据库、连到公司内网

\- 最后把一个任务拆成一串子任务，交给不同子 Agent 去跑

每一步都在烧 Token，每一个 Token 背后都是电费、是 GPU、是数据中心。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zlJYapEUdVPd91NqMDIUepQ081SdvBdWrhibXW5RXEnY7ccg0Ds3DQfN0AmjlwFicOzD6VJOctic2KJYv1icx04tIsuoykUDWO8gSE/640?wx_fmt=png&from=appmsg)

黄仁勋这次直接给出了一个新框架：所谓 Token 工厂经济学。

\- 数据中心不再是“云盘”，是“Token 工厂”

\- 输入是电力、芯片、冷却

\- 输出是可卖钱的 Token：写出来的代码、生成的文案、做完的报表、跑完的模型、执行完的任务

他说得很直白：“未来，每一家云厂、每一家 AI 公司，都会开始算同一笔账：在固定功率之下，每秒能生产多少 Token，每个 Token 能卖多少钱。”

这话如果翻译到普通人的生活里，其实很好理解——你自己的时间，也正在变成一座小型 Token 工厂。

当你一个人写方案，一小时可能只能写 2 页当你有一个靠谱的 Agent，先把资料翻完、梳好结构，你再上去补观点和细节，同样一小时能产出 10 页在用 AI 的人眼里，这不是“效率提升”，而是一种非常现实的算账方式：同样 8 小时，我能比身边人多生产多少“有价值的 Token”。

黄仁勋在台上给出的那个“年薪 + Token 预算”的例子，其实就把这件事说穿了，“未来，每一位工程师都会有一个年度 Token 预算。基础年薪几十万美元，我会额外拿出一半的钱，用来给他买 Token，让他去放大自己的产出。”

这已经不是“配不配电脑”的问题，而是一家公司愿不愿意为“个人算力”和“Agent 劳动力”付费的问题

三、Vera Rubin：给龙虾 Agent 造一台“无限脑容量”的工厂
------------------------------------

很多人看到这一段，可能本能的反应是：推理这么费电，这么烧钱，企业真的能扛得住吗？

这也是为什么，今年 GTC 的技术主角，不是一块单一 GPU，而是整套 Vera Rubin 架构。

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/JYWVQlsu9znE7l8Pwibpp4glopwlKenLDo6kSgN3ycDQ22D26bPn65zDuXCud5URiah8FnlNYKnu9fsxNZUm3grSHa6W0gf7yPFmCe2tOJWpU/640?wx_fmt=png&from=appmsg)

如果用一个最直观的类比，之前的 GPU 像是一台跑分怪兽，训练模型、跑大算子。

而 Vera Rubin，更像一整座为了 Agent 工作流重构过的“算力工厂”：

前端是 Rubin GPU，带着 288GB 级别的大显存，专门装超大模型和长上下文的 KV 缓存；一侧是 Vera CPU，这颗专门为智能体和强化学习设计的 CPU，负责调度工具、跑那些必须串行的逻辑；机架层面，用 Kyber 这种新设计，把 144 张 GPU 接成一个 NVLink 域，把以前复杂的线缆全部藏进去了

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9znFYMRJTicvAFw03UyDOPBGJicE2jfYXNSsYpQJ3B9gAGDe6L1MaUWYeial4ucJV0NFnJONXH4GjMwhIiafl53Arzyu6OeGT3enp9s/640?wx_fmt=png&from=appmsg)

过去的数据中心，更像一个巨型文件柜，现在的数据中心，正在变成一台“Token 汽车厂”：

Rubin GPU 像是冲压车间，负责高密度计算；Vera CPU 像是总装线，人手一堆工具，要把各种外部系统、数据库、服务接口全连起来；中间靠 NVLink 和 Spectrum X 把这些工位焊成一台整体机器

在这种架构下，再叠加上 Groq LPU，这次黄仁勋给出的那条性能曲线，才显得没那么夸张——在最高价值的推理层级上，性能和每瓦吞吐可以拉到 35 倍。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9znsrGXs6emTkTVSNsoEgTib52MSPepusQjR496NPQcuYoo9OnvUWl6hzsibVyaGQ1peRa5dlrpias61YzZJtxtrbtKMCTdbqVNpibM/640?wx_fmt=png&from=appmsg)

背后的逻辑并不复杂：

预填充那一段（Prefill）、注意力那一段，交给 Rubin GPU 这种“重型选手”，生成 Token 那一段（Decode），交给 Groq LPU 这种极致低延迟的小钢炮

对企业来说，这不是纸面上的技术细节，而是一道“算力就是利润”的选择题：如果你的业务更像免费问答、低价值内容生成，就堆 Rubin，把吞吐做到极致；如果你的业务是高客单价交易决策、代码生成、自动运维，那就必须在最贵那一层，把 Token 速度拉满

我看完 Vera Rubin 那几张架构图之后，有一个很强的感觉：英伟达其实是在给龙虾 Agent 这类“新劳动力”建工厂，而不只是单纯在卖显卡。

现在黄仁勋在做的，是把“算力”和“智能体”打包成一个全新的生产要素，然后交给企业去买。

四、NemoClaw：龙虾 Agent 的体制内版本
--------------------------

如果说 Vera Rubin 是地基，那整场演讲真正点燃现场情绪的，其实是另外一只主角——龙虾。

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/JYWVQlsu9zkic6TcS7T6Eh3MicAVqa6IBJZGQlyR6eUibclODviaLWxo4rmlZG8ZpfYPaGEzDiaMDc83Xl92ABgeGlTeEmiagahmSHCwtBwDLMWQU/640?wx_fmt=png&from=appmsg)

这几个月，只要你混过开发者圈或者内容圈，基本都绕不开 OpenClaw：

有人用它当桌面助手，让它帮自己收拾 Mac 上一堆乱七八糟的文件；有人把它接到微信、飞书，帮自己回消息、整理群聊纪要；还有人直接跑起 AI 托管号，让它盯着各个平台的数据、自动调内容节奏。

在深圳，这股热度已经从屏幕里溢到了街上：线下门店里，龙虾周边、定制主机、教程课，像极了当年的“矿机店”和“挖矿群”。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zmNyNLOqydhPwH0fL6KOib86QylSc4jeDvvp9iaqN2LuVNibby7lYE4VmN9RtEQMkrmySicDgTZQh4ibUhbcO8CsFaJNIa9mFSyib8KU/640?wx_fmt=png&from=appmsg)

黄仁勋在 GTC 上讲 OpenClaw，用了非常重的一句话：“OpenClaw 是这个时代的 Linux，是这个时代的 HTML。”

这句话背后有两层意思：

\- Linux 和 HTML，一个把“操作系统”标准化，一个把“网页语言”标准化

\- OpenClaw 想做的，则是把“智能体怎么工作”这件事标准化

从普通用户的视角看，它就是你那只“会干活的龙虾 Agent”：

会看文件、会读代码、会用命令行。会调子 Agent 去干脏活累活，会在后台悄悄执行你半夜设的计划任务。但从企业 CIO 的角度看，问题马上变了：谁来管住这只龙虾？它要是手一抖，把生产库删了怎么办？

这就是 NemoClaw 出场的地方。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zlk0WVQoVvpxPHgrog2tia2fpFibvTXZZ8f7hhVjz8jeYztbQkN2nYnLEibibDU7p2VfQj6bicsboDom7KElgmKBmiaSVCnRBibh3pjq8/640?wx_fmt=png&from=appmsg)

在 OpenClaw 的故事线之外，英伟达拉着“龙虾之父” Peter Steinberger 搭了一套企业级参考架构：

底层是 OpenClaw 那套智能体操作系统，外面包了一层 OpenShell、策略引擎和隐私路由，再往外，是各家企业自己的合规系统、审计系统、安全网关，你可以简单把它理解成——体制内版本的龙虾 Agent。

员工看见的，是一个能帮自己跑流程、写报告、查系统的智能助手；安全部门看见的，是一整套可控的策略引擎和日志体系。

这一点对中国企业来说，其实非常关键：

焦点不在于敢不敢用 OpenClaw，更关键的是有没有一套能被审计、能上报、能备案的智能体基础设施。从这个角度看，OpenClaw 更像民间江湖，NemoClaw 更像官办正规军。

两者合起来，才构成了“龙虾 Agent 时代”的上下半场。

五、龙虾 Agent 要抢谁的饭碗？
------------------

看到这里，很多人都会问同一个问题：龙虾 Agent 到底会先抢谁的饭碗？

黄仁勋在演讲里没有直接给答案，但他提到了一个很微妙的细节：

 “未来，公司里每一位工程师，都会被问一个问题：你一年要多少 Token？你准备用这些 Token 干什么？”

把“工程师”三个字去掉，其实放在任何白领岗位都成立。从这几个月身边人的使用情况看，有几类工作会被 Agent 率先重塑：

1\. 重复度高的表达型工作

日报、周报、汇报 PPT

客户跟进记录、售后话术、运营回复

这些内容，一旦有了足够的历史数据，Agent 很容易学会你的语气、套路，替你打底。

2\. 结构清晰的规则型工作

招投标文件初稿

标准合同审核第一轮

合规条款比对

这类工作，本质上是规则和模板的组合，只要权限给得足够清晰，Agent 甚至比人更能扛住熬夜。

3\. 需要海量检索的分析型工作

行业研究、竞品监测

舆情跟踪、政策解读

以前这些事情要几个人熬几天，现在一台小主机上的龙虾 Agent，配合好数据源，一夜之间就能给你一版还不错的底稿。

这并不意味着所有岗位会一夜消失，更现实的场景是：有人用 Agent 把自己变成一个“站在螃蟹身上的螃蟹”：下面一堆龙虾在帮他干活，他只盯核心判断和拍板也会有人把 Agent 当成“高级搜索”，每天只用它写几句文案、改几段代码，最后发现，是自己的 Token 工厂在原地踏步

这背后有一个比较残酷的变化：公司不再只看你一个人能干多少活，而是看你能调动多少算力、能协调多少智能体。

六、从深圳排队买龙虾，到主权 AI：时代正在加速分层
--------------------------

这次 GTC 里，有一个细节其实很值得中国读者多想一会儿。

黄仁勋在讲 OpenClaw 的时候，专门点名提到深圳：

“人们在深圳排队买龙虾，每个人都在谈论 OpenClaw。”

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zknvbiakiarvNEALg4IpBbNI3jNibmKL2G3Pe2wFAy7alYkibQ1f5h6luRMYycsSDlpNyhfsHFv9ZgK9G5rV8esYBmEriaCAO8QS4b8/640?wx_fmt=png&from=appmsg)

这背后至少有三层意思：

1\. 技术扩散的速度，远超很多人的想象

从硅谷一场开源项目，到深圳街头排队买主机，中间只隔了几个月。

2\. 个人算力正在从“程序员特权”变成一种大众消费品

就像当年的显卡、矿机、云服务器，现在变成了“家用 AI 小机柜”“桌面龙虾工作站”。

3\. 谁先适应 Agent 时代，谁就在下一轮职场洗牌里站得更稳

不管你是个体户、小团队，还是大企业里的中层，现在的选择，其实是在替未来几年自己站在哪条轨道上做决定。

与此同时，在 GTC 的另一条线索上，英伟达已经和多个国家谈起了主权 AI、太空数据中心、Space-1。

一边是深圳街头的龙虾店，一边是绕着地球飞的 AI 工厂，这两个画面放在一起，很难不让人意识到：这轮 AI 浪潮在同时推高三个层级——国家、企业、个人。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9znYpX0RJBPWTWe4JoCxibvJA1a1A9Sp2AlAgCakDzSNzk8fONENaQTLGLzOyZiaZ1vzic5WuKaN8Xyxgn10KDeyfUV5J9CkPgDElk/640?wx_fmt=png&from=appmsg)

国家在抢的是“主权算力”和“AI 工厂选址权”；企业在抢的是“Token 工厂效率”和“Agent 布局速度”；个人在抢的是“能不能用好 Agent”这件事上的时间差。

有人选择观望，等政策更明朗、等产品更成熟；也有人已经悄悄把自己的日常工作拆了一遍，哪些可以外包给 Agent，哪些必须自己亲手做。

从这个意义上说，那句“龙虾 Agent 要来抢你的饭碗了”，不算危言耸听，更像一句很现实的提醒：它到底是抢走你的饭碗，还是帮你多端几个碗，很大程度上取决于你现在怎么用它。

七、普通人的应对建议：别等公司配龙虾，先给自己配
------------------------

说到这里，很多人会问一个更实际的问题：“那我现在应该做什么？”我想用三个很具体、也很接地气的建议收个尾。

### 1\. 先养一只“只替你干一件事”的龙虾

不要指望一上来就让 Agent 接管你全部工作，这样只会让你很快失望。

更稳妥的做法，是先选一件你最讨厌、但又不得不做的重复性工作，给它开一只专门的龙虾。

比如：

\- 把所有日报、周报、月报，统一交给一个“报告龙虾”，让它学会你的话术和模板

\- 把所有会议纪要、录音整理，交给一个“会议龙虾”，只做一件事：帮你保留关键信息

\- 把素材搜集、竞品监测，交给一个“情报龙虾”，每天只给你一页可读的总结

你会很快发现，当一只 Agent 专心只干一件小事时，它的稳定性和可控性，往往比“万能大助手”要靠谱。

### 2\. 把自己的工作拆成“人做的”和“Agent 做的”

这一步，很多人其实是反着来的：

先用上 AI，再来想让它做什么。效果往往一般。更有效的顺序是：

用一张纸把你一周的工作拆开：写、算、沟通、检索、整理、决策，在每一项后面，老老实实打勾：这件事 AI 能不能做，如果能，它做多少，你做多少

然后，把那些“AI 能做 70%，你来收尾 30%”的事情，优先交给 Agent。这一步看起来有点笨，但意义很大。你不是在把自己交给 AI，而是在把 AI 正式纳入你的工作流。

### 3\. 学会算一笔属于自己的“Token 账”

黄仁勋讲“Token 工厂”的时候，是站在企业和数据中心的视角。

但这件事其实可以缩小到个人层面。你完全可以给自己也设一个“小型 Token 预算”：一个月愿意为 AI 工具花多少钱，这些钱能换来多少 Token、多少任务、多少节省下来的时间，这些时间能不能让你多接一个项目、多写一篇深度内容、多学一门新技能

当你开始认真算这笔账的时候，你会发现一个有点扎心的现实：真正贵的往往不是算力，而是你不愿意改变工作方式的那部分惰性。

八、写在最后：把这篇文章当成一个小测试
-------------------

如果你看到这里，可以问问自己三个问题：

1\. 你现在手上有没有一只真正“替你干活”的龙虾 Agent？

2\. 你有没有认真拆过自己的工作，看清哪些可以交给 Agent 去做？

3\. 你有没有开始给自己算一笔“时间 × Token × 收入”的账？

如果答案里有一个“还没有”，现在其实就是一个不错的起点。
