OpenAI 深夜发布 GPT-5.4 mini 和 nano！成本暴降！小模型杀疯了
===========================================

原创 阿竹 AI飞升录 2026-03-18 18:27 浙江

> 原文地址: [https://mp.weixin.qq.com/s/fKpNvEd0XVdx02RzveUW6g](https://mp.weixin.qq.com/s/fKpNvEd0XVdx02RzveUW6g)

凌晨一点多，我刷到一条推送，OpenAI 又放了新模型。

说真的，这半年新模型发布太频繁了，我本来想翻个身继续睡。结果点进去一看定价，一下子清醒了——输出价格只有旗舰版的三分之一，最便宜的那个 nano 更离谱，只有旗舰版的十二分之一。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zlQ7FFicRP1z9c62JW8PscXibqy4UwFGNU9icFMV7GQSaPhdwpxTZBv54pdaSHXAXsFcfhspcBYbxJGy6IYTAia4mT01Vibjibde0nAI/640?wx_fmt=png&from=appmsg)

这次发的两个模型，一个叫 GPT-5.4 mini，一个叫 GPT-5.4 nano。名字听着像是旗舰版的缩水版，但实际跑分出来之后，我只能说：这哪是缩水，这是把旗舰版的功力灌进了一个更小的壳子里。

我把这两个模型的核心信息和我自己的判断整理了一下，争取让你花 5 分钟看完就知道该不该用、怎么用。

先搞清楚：mini 和 nano 到底是干嘛的
-----------------------

一句话讲：mini 是"几乎和旗舰版一样强，但跑得快、花得少"，nano 是"再砍一刀，极致省钱，专门干杂活"。

OpenAI 官方对这两个模型的定位非常明确——给那些需要快速响应、高频调用的场景用的。

什么意思呢？你可以想象一下：

\- 你在用 Cursor 写代码，每按一下 Tab，后台就要调一次模型帮你补全

\- 你在跑一个自动化流程，Agent 每隔几秒就要读一次屏幕截图、判断下一步操作

\- 你的客服系统每分钟要处理几百条用户消息

这些场景下，你不需要一个"思考 30 秒才给你一段完美长文"的大模型。你要的是又快又准，还便宜。

mini 和 nano 就是来干这个的。

跑分到底有多猛
-------

我知道很多人对"跑分"没什么感觉，觉得这些测试名字看着就头大。所以我只挑几个最能说明问题的讲。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zmTMJCLXvHhiaQP1MXvFpAic95FeIFwic8eH4iaIx8gSjo2DBIrZC1bgN8lTPPmzYuenCibAKxcrRx9iaWicSyWb0FkicCtMrPb0ehUCOk/640?wx_fmt=png&from=appmsg)

写代码这一项，有一个叫 SWE-Bench Pro 的测试，专门拿 GitHub 上真实的 Bug 让模型去修。旗舰版 GPT-5.4 得了 57.7%，mini 拿到 54.4%，就差三个多点。而上一代 GPT-5 mini 只有 45.7%。

换句话说，这一代 mini 修 Bug 的能力，已经比上一代 mini 强了快 9 个百分点，而且逼近旗舰版。

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/JYWVQlsu9zlPNJnXSe5xFk23ogpibia1Dxk2W2Gic2cjOia599icBkHb4cRIOIRURvz6eSgUmxBPQKABrOSZavNicbThZPibJFjGbzV29ibBVcFFQso/640?wx_fmt=png&from=appmsg)

再看一个更直观的：计算机操作。就是让 AI 直接看你的电脑屏幕，然后帮你点按钮、填表格、跑流程。

在 OSWorld 这个测试里，旗舰版得了 75%，mini 拿到 72.1%，差距不到 3 个点。而上一代 mini 只有 42%。

这是什么概念？上一代连一半都做不到的事情，这一代直接飙到七成以上。这个进步幅度真的吓人。

还有工具调用。做过 Agent 开发的人应该都知道，模型光能聊天没用，关键是能不能正确地一步步调用外部工具来完成任务。在 Toolathlon 测试里，新款 mini 拿了 42.9%，上一代才 26.9%，提升了接近六成。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zmK3giaHIHHEQC4SXSGQzImO16fYialcK9xUMXgnekI4icCJnAzcjtaubpVLSicKpFQ9icu98FZlspeYjqwBBqbFy9exvm6JLXxqgY4/640?wx_fmt=png&from=appmsg)

nano 的分数整体会低一些，毕竟定位就是最轻量的那个。但即便如此，它在学术推理测试 GPQA Diamond 上还是拿了 82.8%，超过了上一代 mini 的 81.6%。一个 nano，干掉了上一代的 mini，这事本身就够说明问题了。

便宜到什么程度
-------

说完性能，聊聊大家最关心的：到底多少钱。

GPT-5.4 mini 的 API 定价：

\- 输入：每百万 Token 0.75 美元

\- 输出：每百万 Token 4.50 美元

GPT-5.4 nano 的 API 定价：

\- 输入：每百万 Token 0.20 美元

\- 输出：每百万 Token 1.25 美元

做个对比你就明白了：旗舰版 GPT-5.4 的输出价格是 15 美元 / 百万 Token。mini 只要 4.5 美元，nano 只要 1.25 美元。

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/JYWVQlsu9zl9xW9A0vaZSchUDDhQy5ZJLXB9vZ47D5PcIVqp4sk5q4qcL6WawKhdTLTQZ9b6PTIUngj5ibicM9OASV3tTLR427mgqicjvTJf7w/640?wx_fmt=png&from=appmsg)

有个开发者试完之后说了一句话，我觉得挺到位的：它不光快，而且比 Claude 4.6 Opus 便宜 9 倍。

当然也得说句公道话，跟上一代 mini 和 nano 比，价格其实是涨了的。mini 输入价格大约是上一代的 3 倍，nano 输入价格大约是上一代的 4 倍。但考虑到性能的提升幅度，这个涨价大部分人应该能接受。

用一个更生活化的比喻：以前你花 10 块钱买一碗普通面，现在花 15 块买了一碗加了牛肉、溏心蛋还加了辣油的面。涨了 5 块，但吃到嘴里完全两回事。

大脑加手脚：以后 AI 干活的方式要变了
--------------------

这次发布里，我觉得最值得注意的，不是某个跑分数字，而是 OpenAI 在反复强调的一个架构思路——分层调度。

简单说就是：旗舰版 GPT-5.4 当老板，负责想方案、拆任务、做最终判断。mini 当干活的员工，负责并行跑那些具体的子任务。

比如在 Codex 平台上，你可以让 GPT-5.4 先做一个开发计划，然后它会自动把任务拆开，派给好几个 mini 子智能体同时去搜代码库、审代码、查文档。而且 mini 在 Codex 里只消耗旗舰版 30% 的配额，相当于同样的预算能多跑三倍任务。

![图片](https://mmbiz.qpic.cn/mmbiz_png/JYWVQlsu9zlK4BVc64H26eEeuB8Dl9fgZGmiaxxyUN7d45j0XcrprtKcDnpWU8W3dBwIrnM3hqnq8qdya8YXaFhiaBlxibSW7YdOMARlnQWmnk/640?wx_fmt=png&from=appmsg)

这套分工模式，其实跟龙虾 Agent 的逻辑是一脉相承的：大模型当大脑，小模型当手脚。

以前做 Agent 最头疼的问题之一，就是每一步都要调旗舰模型，成本根本扛不住。现在有了一个又快又准又便宜的 mini，很多之前"算不过来账"的 Agent 应用，突然之间就能跑起来了。

说几个短板，省得你踩坑
-----------

当然，这两个模型也不是没弱点。

最明显的短板是长上下文处理。在 OpenAI 自己的 MRCR v2 测试里，64K 到 128K 窗口下，旗舰版得了 86%，mini 只有 47.7%。再拉长到 128K-256K，旗舰版 79.3%，mini 掉到 33.6%。

这意味着如果你的任务需要在一大堆文档里做精确信息检索和逻辑追踪，mini 还是扛不住，老老实实上旗舰版。

nano 的话，计算机操作能力也偏弱，在 OSWorld 测试里只有 39%，比上一代 mini 的 42% 还低一点点。所以如果你想让 AI 帮你操作电脑，nano 不太行，得上 mini。

总结一下选型建议：

\- 写代码、跑 Agent、需要工具调用 → 选 mini

\- 做分类、提取数据、批量处理简单任务 → 选 nano

\- 超长文档精确检索、最高质量推理 → 还是用旗舰版

这事跟普通人有什么关系
-----------

有人可能觉得，这是开发者才需要关心的事情。

其实不完全是。

你每天用的各种 AI 产品，背后都在调模型。模型的成本降一个数量级，意味着这些产品的使用门槛也会跟着降。

举几个很现实的例子：

\- ChatGPT 免费版用户，现在就能用上 mini 的"思考"功能了

\- 付费用户的旗舰版额度用完之后，会自动切换到 mini 继续用，不会断

\- 那些做 AI 客服、AI 写作工具、AI 编程助手的创业公司，推理成本一下子降了大半，他们就有空间把产品做得更好、或者直接降价

从个人角度讲，我这半年最大的一个感受是：AI 能力在涨，但涨得最快的，往往不是顶上那个最贵的旗舰版，而是中间这一层"又够用又便宜"的模型。

对大多数人的日常使用来说，mini 这个级别的能力已经够了。

![图片](https://mmbiz.qpic.cn/sz_mmbiz_png/JYWVQlsu9zkicZXlBIHez84gDAHebWQLd0MjXuK8hXEic22rJib68nibRjAI960uqUyTkj30NiaUTPXicCqZK27lC1DwCYxXsDdWAtrqMZwBFvgwY/640?wx_fmt=png&from=appmsg)

最后说两句我自己的看法
-----------

我翻了一圈评论区，看到有些开发者的心情挺复杂的。一方面这些模型确实好用，另一方面也有人在想：这些模型拿我们写的代码训练，然后又回过头来替代我们写代码，这个事怎么说呢。

这种纠结我能理解。但我觉得更现实的问题是：不管你觉得公不公平，这个方向已经不可逆了。

与其去想"它会不会替代我"，不如想想怎么让它替你干活。

前面讲的那个"大脑 + 手脚"的架构，其实也可以套到个人身上：你当大脑，mini 和 nano 当你的手脚。把那些你最不想干、又不得不干的活交出去，你腾出来的时间做更有价值的判断和决策。

这一轮小模型的爆发，我觉得真正的信号是：AI 的门槛正在加速往下降。以前玩得起 AI 的是大厂，后来是中小企业，现在正在轮到个人。

谁先学会用这些工具，谁就能比身边的人多出好几双手。
