$0搭服务器，甲骨文4核24G白嫖指南
===================

甲骨文（Oracle Cloud）给每个注册用户一台永久免费的 ARM 服务器。4核CPU、24GB内存、200GB存储、每月10TB流量，不限期，不扣钱。

同样的配置在 AWS 上月费大约 $80-100。

配置水平
----

核心配置：

•

4核 Ampere A1（ARM 架构）

•

24GB 内存

•

200GB 硬盘

•

每月 10TB 出站流量

•

永久免费，不限期

甲骨文按小时算配额：每月 3,000 OCPU hours + 18,000 GB-hours。一台 4核/24G 刚好吃满，所以免费账号最多拉满一台 ARM 实例。

除了主力 ARM 机器，还附赠 2 台 AMD x86 小机器，各约 1/8 核/1GB，配置弱但能跑轻量任务。200G 存储可以灵活分配，大多数人直接一台拉满最省事。

### 能干什么

搭个人网站（WordPress 或 Hugo），跑 AI 推理（Ollama 支持 ARM），建个人云盘（Nextcloud 或 Alist），挂 Bot 或定时爬虫，开 Docker 容器练手，装个数据库给小项目当后端。

4核24G 跑一套 Nginx + PostgreSQL + Redis 富余得很，有用户内存常年占不到 8G。

注册准备
----

### 三样东西

**实体信用卡。**银行发的 Visa 或 Mastercard，招行工行中行浦发都行。虚拟卡、预付卡、礼品卡不认。绑卡扣大约 $1 做验证，几天内自动退回。

**邮箱 + 手机号。**国内邮箱和 +86 手机号直接能用。

**手机热点。**注册时关掉所有 VPN 和代理，用手机 4G/5G 热点连电脑，别用 WiFi。

### 为什么用热点

甲骨文风控会核对三样东西：当前 IP 所在国、信用卡发卡国、账单地址国。用中国信用卡注册，IP 必须来自中国。

公司 WiFi 或开了代理，IP 可能飘到别的国家去。有人在公司网络下试了三次都被拒，换成手机热点一次就过了。

### 踩坑排查

提交后出现红色感叹号，先查这几项：

•

VPN / 代理是不是完全关掉了？浏览器插件也算

•

用的是手机热点还是 WiFi？

•

信用卡是实体 Visa / Mastercard 吗？

•

账单地址填的是中国地址吗？

•

同一张卡试了 2 次以上？停手，换卡或等 2-3 天

核心就一条：IP国、信用卡发行国、账单地址国三者一致。同一张卡别反复试，有人试了五六次卡直接被风控标记了。

区域选择
----

注册第 5 步选「主区域」，选完永远改不了，直接决定后面能不能抢到 ARM 实例。

推荐选  **Phoenix（凤凰城）** 免费用户扎堆抢 ARM，热门区域根本抢不到。Phoenix 相对冷门，容量充足，成功率最高。

Mumbai（孟买）和 Jakarta（雅加达）也能选，但不如 Phoenix 稳。Tokyo、Seoul、Singapore 免费用户太多，不建议。

有人担心 Phoenix 离中国远延迟高。服务器不是打游戏，延迟没那么重要，跑个反向代理或挂个 CDN 就能解决。

建实例
---

注册成功后进控制台，Compute → Instances → Create Instance。

配置这样填：

•

镜像选 Ubuntu 22.04，认准标有 Always Free Eligible 的

•

形状选 VM.Standard.A1.Flex

•

OCPU 设 4，Memory 设 24 GB

•

引导卷自定义大小，设 200 GB

•

上传 SSH 公钥

### SSH 密钥

没有密钥文件的话，本地终端跑：

    ssh-keygen -t ed25519 -f ~/.ssh/oracle_key

Windows 用 Windows Terminal 或 PuTTYgen 都行。生成的  `oracle_key.pub`  上传到控制台。

点 Create，等几分钟变绿就是跑起来了。

![oracle_instance_detail](https://mmbiz.qpic.cn/mmbiz_jpg/JYWVQlsu9zmRzHhkhckicfqPaDPUkZwKEorliaFZVSThNsJOK5DFobkaYiaeljhBAXFRgAibOKnDt9bib8J8TO7wiadxmOe4nndKqCExgfDia7sROg/640?wx_fmt=jpeg&from=appmsg)

拿到公网 IP 后 SSH 登进去：

    ssh -i ~/.ssh/oracle_key ubuntu@<你的公网IP>

默认用户名  `ubuntu` 。如果想用密码登录（方便调试），创建实例时在初始化脚本里加：

    #!/bin/bashecho root:你的密码 | sudo chpasswd rootsudo sed -i 's/^#?PermitRootLogin.*/PermitRootLogin yes/g' /etc/ssh/sshd_configsudo sed -i 's/^#?PasswordAuthentication.*/PasswordAuthentication yes/g' /etc/ssh/sshd_configsudo service sshd restart

有个坑：改了 sshd\_config 还是不通的话，去检查  `/etc/ssh/sshd_config.d/60-cloudimg-settings.conf` ，这个文件会覆盖主配置，把  `PasswordAuthentication`  改成  `yes` 。不少人在这卡了半天。

GitHub Actions 版，放 GitHub 自动跑不用开电脑：github.com/maoucodes/oci-free-arm-instance

•

Python 版，本地跑，能配 Telegram 通知：github.com/mohankumarpaluru/oracle-freetier-instance-creation

•

PHP 版，社区最老牌：github.com/hitrov/oci-arm-host-capacity

通常几小时到几天能抢到，高峰期可能更久。

初始化
---

机器拿到后，先做这几步。

### 清防火墙

甲骨文有个很坑的设计：实例内置 iptables，控制台还有安全列表，两层防火墙同时生效。不少人折腾了半天 nginx，最后发现是 iptables 挡着。

直接清掉 iptables，只用控制台的安全列表：

    sudo iptables -Fsudo iptables -P INPUT ACCEPTsudo iptables -P FORWARD ACCEPTsudo iptables -P OUTPUT ACCEPT

然后在控制台 Networking → Virtual Cloud Networks → Security Lists 加端口：22（SSH）、80（HTTP）、443（HTTPS），后面用到再加。

### 装软件

    sudo apt update && sudo apt upgrade -ysudo apt install -y docker.io docker-compose git nginx certbot

不熟命令行的话，装个 1Panel（国产轻量）或 aapanel（宝塔国际版），图形化管理方便很多。

### x86 加 Swap

那两台 AMD x86 小机器只有 1G 内存，跑几个服务就卡死。加 2G Swap：

    sudo fallocate -l 2G /swapfilesudochmod 600 /swapfilesudo mkswap /swapfilesudo swapon /swapfileecho'/swapfile none swap sw 0 0' | sudotee -a /etc/fstab

### 挂硬盘

引导卷之外又加了块存储的话，需要手动挂：

    fdisk -lfdisk /dev/sdb                # 按 n → p → 回车 → w 分区mkfs.ext4 /dev/sdb1mkdir /datamount /dev/sdb1 /dataecho'/dev/sdb1 /data ext4 defaults 0 0' >> /etc/fstab

保活
--

甲骨文会回收闲置机器。连续 7 天，CPU 利用率 95% 的时间低于 10%、网络低于 10%、内存低于 10%（仅 ARM），就可能被收走。历史上阈值调过，以官方最新为准。

**最稳的办法：跑真实服务。**  部署一个真正要用的东西，有真实流量就不用担心保活。有人那台跑了快两年，从来没操过这个心。

还没想好跑什么的话，先用 NeverIdle（github.com/layou233/NeverIdle）顶着。它会定时占用 CPU、内存和网络三项，覆盖甲骨文的回收检测。

三个参数： `-c`  是 CPU 定时消耗间隔（比如  `2h30m` ）， `-m`  是占用内存量（比如  `2`  表示占 2GB）， `-n`  是网络定时消耗间隔。

不想装额外软件，crontab 也行：

    crontab -e# 每 5 分钟打一次请求*/5 * * * * curl -s https://www.oracle.com > /dev/null

![oracle_cron_keepalive](https://mmbiz.qpic.cn/mmbiz_jpg/JYWVQlsu9zkj7icuicrUWetiaKcYPekgDuWYtMPGNgS7jGmAu00g2lOE2TLjXibVCFKnkXcn5Qgs7ibmc2qRKoB6o5PsS0BLCkPPjSgCaBYlc6Iw/640?wx_fmt=jpeg&from=appmsg)

保活脚本有失效风险，甲骨文改检测机制就可能挂。跑起真实服务后就不需要这些了。

常见问题
----

**Q: 甲骨文免费账号会被收费吗？**  
A: Always Free 资源不会产生费用。别手动开付费服务就行。在 Billing 页面设个预算告警，防意外扣费。

**Q: 一张信用卡能注册几个账号？**  
A: 一个。一张卡只能绑一个甲骨文账号。

**Q: 注册选错了区域怎么办？**  
A: 改不了。只能注销账号，等段时间用同一张卡重新注册（有风险，建议第一次就选对）。

**Q: ARM 实例和 AMD 实例有啥区别？**  
A: ARM（Ampere A1）配置高（4核/24G），是免费套餐的主力。AMD x86 配置低（约1/8核/1G），但兼容性好，适合跑不支持 ARM 的软件。

**Q: 能用来做商业项目吗？**  
A: 能。甲骨文对 Always Free 资源不限制用途。但免费实例没 SLA 保障，对可用性要求高的生产环境不太合适。

**Q: 服务器被回收了还能再申请吗？**  
A: 能。账号还在就能重新创建 ARM 实例（有容量的话）。所以保活别偷懒。

**Q: 用 Cloudflare WARP 装代理要注意什么？**  
A: 必须先  `warp-cli mode proxy`  切换成 SOCKS5 模式再  `warp-cli connect` 。如果直接连，WARP 会接管全部流量导致 SSH 断联，只能重装系统。
