小龙虾是什么？为啥这么火？
-------------

**OpenClaw**（国内叫「小龙虾」）是 2025 年底横空出世的开源 AI 项目，一上线就破了 GitHub 历史纪录：

*   **25 万+ Stars**，4 个月达成——Linux 用了 12 年，React 用了 8 年
    
*   **2 天破 10 万 Star**，史上最快
    
*   一度登顶 GitHub 最受欢迎开源项目
    

为啥火？因为它不是「只能聊天的 AI」，而是**能真帮你干活的 AI 助理**——跑在你自己的电脑上，数据不出家门，MIT 开源免费。

### 小龙虾能干啥？

你可以把它理解成**住在你电脑里的全能实习生**：你发一句话，它自己去查、去写、去执行。

场景

你一句话

它自动做

日报

「帮我整理今天的日报，从项目群提取进展」

读群消息 → 提炼 → 按格式生成 → 发到文档

查资料

「搜索 2025 年 AI 编程助手市场规模，整理成表格」

搜网页 → 提取信息 → 做成表格 → 标注来源

日常杂活

「把这份 PDF 转成 Markdown」「翻译这个文档」

直接处理，结果发回给你

还能接 **WhatsApp、Telegram、飞书、Discord** 等 15+ 平台，在聊天里 @ 它就能办事。支持 **Claude、GPT、KIMI、MiniMax** 等主流大模型，你选哪家就用哪家。

**一句话**：ChatGPT 只能聊，小龙虾能干活。装在你电脑上，7×24 待命。

想用小龙虾当你的私人 AI，又怕要写代码、看不懂英文？

别慌。这篇教程参考了多份实测文档，**每一步都可复制粘贴**，照着做就能把小龙虾接回家。Mac、Windows、Linux 三种系统全都有，还有全中文界面版本。

零、开始前：先准备这两样（强烈建议）
------------------

实测经验：**先准备好再装，成功率更高。**

### 1\. Node.js ≥ 22

打开终端（Mac/Linux）或 PowerShell（Windows），输入：

`node -v   `

*   显示 `v22.x` 或更高 → 可以继续
    
*   显示 `command not found` 或版本低于 22 → 去 https://nodejs.org 下载 LTS 版安装（Windows 也可用 `winget install OpenJS.NodeJS.LTS`）
    

**注意**：装完 Node 后，**关掉终端再开一个新的**，否则可能还识别不到。

### 2\. API Key（建议先拿好）

装完小龙虾后，向导会要你选模型、填 Key。**提前拿好 Key，一次跑通。**

六、API Key 还没配？补配方法
------------------

如果 onboard 时跳过了 API，后面可以补：

`openclaw onboard   `

向导会让你选厂商、粘贴 Key。**粘贴时终端不显示是正常的**，直接粘贴后回车。

七、常见问题与报错
---------

### 7.1 安装阶段

报错/现象

解决方法

`openclaw: command not found`

1\. 关掉终端重开  
2\. 执行 `source ~/.zshrc`（Mac）或重启 PowerShell（Win）  
3\. Mac/Linux：`echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc`  
4\. 或运行 `openclaw doctor`

`SyntaxError`

 或 `fetch is not a function`

Node 版本太低，去 https://nodejs.org 下载 LTS，装完重启终端

`EACCES: permission denied`

Linux/Mac：`mkdir ~/.npm-global` → `npm config set prefix '~/.npm-global'` → `echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc` → `source ~/.bashrc`

安装脚本卡住

检查网络/代理；Mac/Linux 可试：`curl -fsSL https://openclaw.ai/install.sh | bash -s -- --verbose`

Windows：PowerShell 拒绝运行脚本

`Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`

Windows：`openclaw 不是内部或外部命令`

1\. 运行 `npm prefix -g`，记下返回路径（如 `C:\Users\xxx\AppData\Roaming\npm`）  
2\. 右键「此电脑」→ 属性 → 高级系统设置 → 环境变量 → 用户变量里找到 Path → 编辑 → 新建 → 粘贴该路径 → 确定  
3\. 关掉所有 PowerShell 窗口，重新打开再试；或改用中文版脚本

### 7.2 配置与启动

报错/现象

解决方法

`Gateway failed to start`

端口被占用。Mac/Linux：`lsof -i :18789`；Win：`netstat -ano | findstr 18789`。先 `openclaw gateway stop`，再 `openclaw gateway start`

`another gateway instance is already listening`

先 `openclaw gateway stop`，再 `openclaw gateway start`

浏览器 127.0.0.1:18789 连不上

1. `openclaw gateway status` 看状态  
2. `openclaw gateway restart`  
3\. 检查防火墙是否拦截 18789

发消息一直「正在输入」没回复

1\. 检查 API Key 是否配对  
2\. 运行 `openclaw onboard` 重新配 Key  
3\. 检查网络能否访问对应 AI 服务商（国内：OpenClaw 本身不需翻墙，但 KIMI/MiniMax 等需能访问）
