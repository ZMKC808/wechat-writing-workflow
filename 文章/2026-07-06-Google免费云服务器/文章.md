# Google这台免费云服务器，被我薅明白了

之前写过甲骨文和 AWS 的云服务器白嫖指南。

甲骨文配置确实香，4 核 24G，200G 硬盘，每月 10TB 流量，放在免费服务器里属于犯规级别。但问题也明显：注册容易被风控，ARM 实例还得抢，很多人卡在第一步。

AWS 顺滑很多，教程多、生态全，适合新手入门。但它更像一套新用户试用权益，时间和抵扣金用完之后，就得重新算账。

后台一直有人问：还有没有第三个选择？

有，Google Cloud。

Google Cloud 也有 Always Free 配额，可以开一台长期在线的小服务器。

它最适合当工具机：挂轻量脚本、跑小网站、做监控、练 Linux、放一个 Docker 小服务。

不是靠配置碾压，而是胜在稳定、规则清楚、控制台成熟。对很多人来说，能长期放着不折腾，比纸面参数更重要。

关键是创建时几个地方要按免费配额来选：区域、机型、磁盘、备份、网络层。

下面全程照着点。

## 先把免费边界讲明白

Google Cloud 的免费服务器，吃的是一组具体的 Always Free 配额。

这次按下面这个组合来开：

- 区域放在 `us-west1`、`us-central1`、`us-east1`
- 机器用 `e2-micro`
- 磁盘要选 `Standard persistent disk`
- 每月标准持久磁盘免费额度是 30GB-month
- GPU、TPU、额外磁盘、负载均衡、Cloud NAT 这些先别加
- 备份、快照、部分监控组件也可能单独收费

最容易扣费的地方，不是机器本身。

而是你创建实例时默认点错了：区域选错、磁盘选成 Balanced、备份没关、网络层没改。

所以这篇的重点很简单：把免费组合一次配对，先拿到一台能长期跑的小机器。

## 准备工作

需要三样东西：

- Google 账号
- 能验证的信用卡或借记卡
- 能正常打开 Google Cloud Console 的网络环境

新用户一般会看到 Google Cloud Free Trial，官方现在是 90 天、300 美元试用金。绑卡会有一个很小的临时验证，不等于正式扣费。

注意，试用金和 Always Free 是两回事。

试用金是给你试用付费服务的；Always Free 是长期免费配额。我们这篇要做的是后者。

## 创建实例

进 Google Cloud Console。

路径是：

`Compute Engine` -> `VM instances` -> `Create instance`

名称随便写，比如 `free-google-vm`。

## 第一个关键：区域认准这 3 个

Region 放在下面三个里：

- `us-west1`，Oregon
- `us-central1`，Iowa
- `us-east1`，South Carolina

其它区域就算也能创建 `e2-micro`，也不算 Always Free。

新手直接选 `us-central1 (Iowa)` 就行。Zone 可以先选 Any，让 Google 自己分配。

![[图片/01-region-us-central1.jpg]]

这一步别为了离中国近选东京、新加坡、香港。

Google 的免费实例不在这些地方。延迟低一点没用，账单高起来才是真的疼。

## 第二个关键：机器认准 E2 + e2-micro

Machine configuration 里，Series 选 `E2`。

Machine type 选 `e2-micro`。

![[图片/02-e2-micro.jpg]]

它显示是 2 vCPU、1GB 内存，但这是共享核心，不是给你独占两颗 CPU。

能跑轻量服务，不能当高性能机器用。

适合做这些：

- 静态网站、小博客
- RSS、监控、签到脚本
- Telegram / Discord Bot
- Tailscale / ZeroTier 节点
- 轻量 Docker 服务
- Linux 和云服务器练手

不适合做这些：

- 大流量代理
- 视频转码
- 数据库重活
- AI 推理
- 高并发网站
- 大规模爬虫

它的定位就是一台“长期在线的小工具机”。

## 第三个关键：磁盘必须改成 Standard

左侧切到 `OS and storage`。

点 Boot disk 下面的 `Change`。

这里有个坑：Google Cloud 现在经常默认给你选更好的磁盘，比如 Balanced persistent disk。

Balanced 听起来只是“均衡”，但它不是免费配额里的标准磁盘。

要手动改成：

`Standard persistent disk`

大小写 10GB 就够了。

![[图片/03-standard-persistent-disk.jpg]]

官方免费额度是 30GB-month，但我建议新手先写 10GB。

不是因为 30GB 不免费，而是你现在只是拿它跑轻量服务，10GB 更容易核对账单。后面真不够，再扩也来得及。

系统镜像选 Debian 或 Ubuntu 都行。教程最多的是 Ubuntu，省心一点。

## 第四个关键：备份关掉

左侧切到 `Data protection`。

Backup 选择：

`No backups`

这一步很重要。

备份、快照不是白送的。你机器本身可能在免费额度里，但快照存储会作为另一个 SKU 出现在账单里。

如果你现在只是测试和练手，先不要开自动备份。

等真的放了重要服务，再单独设计备份策略。

## 第五个关键：网络层改成 Standard

左侧切到 `Networking`。

展开 `Network interfaces`。

找到 `Network Service Tier`。

默认通常是 Premium，手动改成：

`Standard`

![[图片/04-standard-network-tier.jpg]]

这里要说清楚一点。

Google Compute Engine Always Free 官方基础出站流量只有每月 1GB。

但 Google Cloud 的 VPC Standard Tier 还有一个免费项：Standard Tier 每月有 200GiB 免费用量。你截图里的“200GB / mo free”说的是这个。

所以自用轻量服务，选 Standard 基本够用。

但别理解成“Google Cloud 随便 200G 免费”。你开错网络层、跑大流量、跨区域折腾，账单照样会来。

## 第六个关键：保持干净配置

创建实例页面很长，Google 会给你很多看起来很专业的选项。

新手先保持干净配置：不加额外硬件，不开快照备份，不建负载均衡和 NAT，不保留静态公网 IP，也不要为了“更快”改回 Premium network。

云厂商最容易让新手破防的地方，就是每个小按钮背后都有一个计费项。

先用最干净的配置把机器跑起来，再说后面的事。

## 创建之后怎么连

配置确认完，点 `Create`。

等实例变成 Running。

Google Cloud 控制台里可以直接点 SSH，浏览器会打开一个终端。

如果你习惯本地 SSH，也可以在实例详情里复制 External IP，然后用本地终端连接。

Debian 默认用户名一般是你创建实例时的用户名或控制台生成的用户；Ubuntu 常见是 `ubuntu`。Google Cloud 的浏览器 SSH 对新手最省事，先用它就行。

## 拿到机器后，先做一次初始化

如果你用的是 Debian，可以先换成国内源。这样装软件更稳，也减少去一些海外 CDN 拉包的概率。

```bash
sudo bash -c '\
rm -rf /etc/apt/sources.list.d/* && \
truncate -s 0 /etc/apt/sources.list && \
cat > /etc/apt/sources.list <<EOF
deb http://mirrors.aliyun.com/debian/ bookworm main contrib non-free non-free-firmware
deb http://mirrors.aliyun.com/debian/ bookworm-updates main contrib non-free non-free-firmware
deb http://mirrors.aliyun.com/debian-security bookworm-security main contrib non-free non-free-firmware
deb http://mirrors.aliyun.com/debian/ bookworm-backports main contrib non-free non-free-firmware
EOF
apt update'
```

上面这段只适合 Debian 12，也就是 `bookworm`。如果你选的是 Ubuntu，别直接复制，Ubuntu 要换自己的源。

然后装几个基础工具：

```bash
sudo apt install -y iptables ipset curl git nginx
```

如果你想切到 root 用户，可以设置 root 密码：

```bash
sudo passwd
su -
```

但我不建议新手默认打开 root SSH 登录。平时用 Google Cloud 控制台 SSH 或普通用户登录就够了。

如果你明确需要 root 远程登录，再执行：

```bash
sudo sed -i 's/^#*PermitRootLogin.*/PermitRootLogin yes/' /etc/ssh/sshd_config
sudo systemctl restart ssh
```

还有一个进阶操作：用 `iptables + ipset` 拦截常见 CDN IP 段，减少服务走到 Cloudflare、Fastly、Akamai 这些 CDN 的概率。

```bash
bash <(curl -fsSL https://quzei.com/sh/cdn_block.sh)
```

这个脚本会下载一份 CDN IP 列表，写入 `ipset`，然后用 `iptables` 拦截这些 IP 的入站和出站。

它适合明确想控制 CDN 流量的人，不适合所有人无脑执行。因为拦截后，某些网站、镜像源、Docker 拉取、API 请求可能会访问失败。

想装 Docker 的话，再装：

```bash
sudo apt install -y docker.io docker-compose
sudo systemctl enable --now docker
```

这台机器适合跑轻量服务，别一上来就装一堆面板。

真要图形化管理，1Panel、宝塔国际版这种可以试，但服务别开太满。

## 第二天一定看账单

机器创建完，不代表你就可以忘了它。

第二天打开：

`Billing` -> `Reports`

看有没有 Compute Engine 相关费用。

如果有小额费用，优先排查：

- Region 是不是选错了
- Machine type 是不是 `e2-micro`
- Boot disk 是不是 `Standard persistent disk`
- 磁盘是不是超过 30GB
- 是否开了备份或快照
- Network Service Tier 是否还是 Premium
- 是否开了第二台机器
- 是否保留了静态 IP
- 是否创建了额外的网络、NAT、负载均衡

账单不是洪水猛兽。

真正危险的是你以为免费，结果一个月以后才第一次看账单。

云服务器刚创建的前两天，每天看一眼 Billing，基本能把坑堵住。

## 和甲骨文、AWS 怎么选

甲骨文最香，4 核 24G，200G 硬盘，免费配置吊打 Google 和 AWS。

但甲骨文的问题是门槛高：注册风控、区域选择、ARM 容量、抢实例，任何一步都可能卡住。

AWS 最适合新手入门。

注册和控制台体验顺，教程多，生态最完整。但免费权益更偏试用，新用户福利用完以后要重新算成本。

Google Cloud 这台最弱。

但它胜在规则清楚、控制台成熟、创建稳定。你照着免费配置卡住，它就是一台长期在线的小机器。

我的建议：

如果你想要高配置，先折腾甲骨文。

如果你是纯新手，先从 AWS 或 Google Cloud 开始。

如果你已经有一台主力服务器，Google 这台很适合当备用节点、监控机、脚本机。

反正都不冲突。

## 最后给一张防扣费清单

创建前主要核对 5 件事：

- 地区在 `us-west1` / `us-central1` / `us-east1`
- 机型是 `E2` + `e2-micro`
- 磁盘是 `Standard persistent disk`，新手先 10GB
- 备份和快照先不开
- 网络层是 `Standard`，第二天记得看 Billing

照这个清单走，基本就能拿到一台长期免费的 Google 云服务器。

它不是大机器，但能替你长期跑很多小事。

真正重要的是先拥有一台可以随便折腾的机器。

等你把 SSH、Docker、Nginx、域名、证书这些都跑通，再去考虑更大的服务器。

别一开始就追求配置。
